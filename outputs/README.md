# Outputs Directory

This directory contains all analysis outputs generated by the EpiAgent Skills workflows.

## Directory Structure

```
outputs/
├── plots/              # Visualizations (PNG, PDF)
│   ├── epicurve.png
│   └── cfr_rolling.png
└── tables/             # Analysis results (CSV, XLSX)
    └── cfr_rolling_estimates.csv
```

## Plots (`plots/`)

Publication-ready visualizations created using `ggplot2` and `tracetheme`.

### `epicurve.png`
- **Type**: Epidemic curve (histogram)
- **Dimensions**: 8" × 5" at 300 DPI
- **Format**: PNG with white background
- **Content**: Weekly case counts by date of symptom onset
- **Usage**: Shows temporal distribution of outbreak
- **Created by**: `create_visualizations()` function

**Interpretation**:
- Peak indicates maximum transmission period
- Shape reveals outbreak dynamics (explosive vs sustained)
- Duration shows outbreak timeline

### `cfr_rolling.png`
- **Type**: Time series with confidence intervals
- **Dimensions**: 8" × 5" at 300 DPI
- **Format**: PNG with white background
- **Content**: Rolling case fatality risk estimates over time
- **Features**:
  - Point estimate (solid line)
  - 95% confidence interval (shaded ribbon)
  - Percentage scale (0-100%)
- **Created by**: `create_visualizations()` function

**Interpretation**:
- Early estimates have wide uncertainty (fewer data)
- CFR may change over time due to:
  - Improved treatment
  - Healthcare capacity
  - Case detection changes
  - Virus evolution

### Customizing Plots

#### Change Dimensions
```r
ggsave(
  here::here("outputs", "plots", "epicurve.png"),
  plot = epicurve_plot,
  width = 10,   # Wider figure
  height = 6,   # Taller figure
  dpi = 300
)
```

#### Different Format
```r
# Vector format for publications
ggsave(
  here::here("outputs", "plots", "epicurve.pdf"),
  plot = epicurve_plot,
  width = 7, height = 5,
  device = "pdf"
)

# High-resolution for posters
ggsave(
  here::here("outputs", "plots", "epicurve_poster.png"),
  width = 12, height = 8,
  dpi = 600  # Higher resolution
)
```

## Tables (`tables/`)

Analysis results in tabular format for further processing or reporting.

### `cfr_rolling_estimates.csv`
- **Format**: CSV (comma-separated values)
- **Encoding**: UTF-8
- **Rows**: One per date in outbreak period
- **Created by**: `estimate_cfr()` function

**Columns**:
- `date`: Date (YYYY-MM-DD format)
- `severity_estimate`: Point estimate of CFR (0-1 scale)
- `severity_low`: Lower bound of 95% CI
- `severity_high`: Upper bound of 95% CI
- Additional columns may include:
  - `cases`: Cumulative cases
  - `deaths`: Cumulative deaths

**Usage**:
```r
# Read the results
library(readr)
cfr_data <- read_csv(here::here("outputs", "tables", "cfr_rolling_estimates.csv"))

# Quick summary
summary(cfr_data$severity_estimate)

# Plot in different software
# Import cfr_rolling_estimates.csv into Excel, Python, etc.
```

**Interpretation**:
- Early dates: High uncertainty, fewer observations
- Later dates: More stable estimates
- Confidence intervals: Wider = more uncertainty

## Output Specifications

### Figure Standards

All figures follow these specifications:

| Aspect | Specification | Rationale |
|--------|---------------|-----------|
| **Resolution** | 300 DPI | Publication quality |
| **Format** | PNG (primary), PDF (vector) | Widely compatible |
| **Background** | White | Print-friendly |
| **Fonts** | Sans-serif, ≥9pt | Legible when scaled |
| **Colors** | Colorblind-safe | Accessible |
| **Theme** | `tracetheme::theme_trace()` | Consistent branding |

### Table Standards

| Aspect | Specification |
|--------|---------------|
| **Format** | CSV (UTF-8) |
| **Delimiter** | Comma |
| **Date Format** | ISO 8601 (YYYY-MM-DD) |
| **Decimal** | Period (.) |
| **Missing Values** | NA |
| **Header** | First row |

## File Naming Conventions

Follow these patterns for consistency:

**Plots**:
- Descriptive name: `epicurve.png`, `cfr_rolling.png`
- Date-stamped: `epicurve_2024-01-15.png`
- Versioned: `analysis_v1.png`, `analysis_v2.png`

**Tables**:
- Analysis type: `cfr_estimates.csv`, `rt_estimates.csv`
- With parameters: `cfr_delay7_window14.csv`
- Timestamped: `results_2024-01-15_14-30.csv`

## Reproducibility

### Regenerating Outputs

To regenerate all outputs with the same parameters:

```r
# Run complete workflow
source("ebola_workflow.R")

# Outputs will be overwritten in outputs/
```

### Archiving Outputs

For important analyses, archive outputs with timestamps:

```bash
# Create archive directory
mkdir -p outputs/archive/2024-01-15

# Copy current outputs
cp outputs/plots/*.png outputs/archive/2024-01-15/
cp outputs/tables/*.csv outputs/archive/2024-01-15/

# Or create timestamped archive
tar -czf outputs/archive/analysis_2024-01-15.tar.gz outputs/plots outputs/tables
```

## Quality Checks

Before considering outputs final:

- [ ] Figures render correctly (no clipping, legible text)

- [ ] File names are descriptive and follow conventions

## File Management

### Cleanup Old Outputs
```bash
# Remove all outputs (keeps directory structure)
rm outputs/plots/*.png
rm outputs/plots/*.pdf
rm outputs/tables/*.csv

# Selective cleanup (keep recent)
find outputs/plots -name "*.png" -mtime +30 -delete  # Delete >30 days old
```

### Disk Space Management

Typical output sizes:
- PNG plots: 50-200 KB each
- CSV tables: 10-100 KB each
- Total for standard workflow: <1 MB

For large-scale analyses:
- Archive old outputs regularly
- Use compression for storage
- Consider reducing plot DPI for drafts (150 DPI)

## Troubleshooting

### Plots Not Generating
```r
# Check if directory exists
dir.exists(here::here("outputs", "plots"))

# Create if missing
dir.create(here::here("outputs", "plots"), recursive = TRUE)

# Check write permissions
file.access(here::here("outputs", "plots"), mode = 2)  # 0 = writeable
```

### Tables Empty or Incorrect
```r
# Verify data before saving
print(head(cfr_results$rolling))
print(nrow(cfr_results$rolling))

# Check for NA values
sum(is.na(cfr_results$rolling))
```

### File Size Issues
```r
# Check plot size
file.info(here::here("outputs", "plots", "epicurve.png"))$size / 1024  # KB

# Reduce file size by adjusting DPI
ggsave(..., dpi = 150)  # Instead of 300 for drafts
```
